Page plus its data. One zip, right layout.

window.exporter emits a page and the data it read()s as one archive — HTML at the root, each read value frozen to its local form beside it — so unzip-and-open resolves local with no manual placement. By default it's local-data (the saved page still boots its code from the CDN); { offline: true } bakes the code in too, for a page that opens with no network at all.

Colophon

Driven by demo.mount(…) from vanilla-demo.js. The parent kind runs from the Run click so the download gets a genuine user gesture. The page-and-data integration this kit was built for lives in cross-repo-read-demo.html.

Unzip it. See how Excel wires it together.

window.xlsxKit unzips an .xlsx, walks every XML/rels part, and returns the internal cross-references (shared strings, styles, sheet rels, comments, calc chain, defined names) plus reconstructed sheet data — no DOM, no Tabulator, just plain objects. Needs a real file, so this uses the Console · host kind — the Run click is the gesture behind the file dialog.

Test cases


    
    

      
1. Basic counter (internal x-data)

      

        Expected: a button labeled "Count: 0" that increments on click. If the box
        renders empty, the host element exists but its template body never landed.
      

      

        
      

    


    
    

      
2. $attrs reactive read

      

        Expected: two greetings — "Hello, Ada! (start=42)" and "Hello, Grace! (start=7)".
      

      

        
        
      

      

        
        — $attrs should react via the MutationObserver.
      

    


    
    

      
3. Static template (no inner x-data)

      

        Expected: the template body renders even without an inner x-data block.
        If this case works but case 1 doesn't, the issue is with Alpine.initTree of
        the cloned content, not with the upgrade itself.
      

      

        
      

    


    
    

      
4. External Alpine.$data() reach-in

      

        Expected: clicking "+1 from outside" increments the inner counter. Verifies
        that the cloned content's x-data scope was actually initialized.
      

      

        
        
        
      

    


    
    

      
5. Defined while nested inside x-data

      

        Expected: the directive warns to the console but still registers the element.
        Look for the warning in the Diagnostics → Console capture panel above.
      

      

        
      

    


    
    

      
6. Invalid name (no hyphen)

      

        Expected: the directive warns and does NOT register. The tag below should
        remain an HTMLUnknownElement and the box stays empty.

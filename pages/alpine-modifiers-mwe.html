what's actually happening

For each element, the scanner reads attributes whose name starts with x-, splits the name on dots to extract the directive and modifiers, and calls the registered callback with { el, modifiers, expression }. The mapping from modifier strings to CSS classes is hand-coded inside each directive — no proxy, no Alpine internals. This is the same pattern Alpine uses; Alpine just adds reactivity, lifecycle, and a much bigger built-in directive set on top of it.

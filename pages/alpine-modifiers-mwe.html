The directive callback gets { modifiers, expression } — modifiers is just an array of strings parsed from the dot-separated attribute name; expression is the attribute value, evaluated against the surrounding Alpine scope on demand. The mapping from modifier strings to daisyUI classes lives inside each directive's body — hand-coded, no proxy, no magic. Same logic we'd have written inside fills.btn(['xs','primary'], …, 'count++'), just delivered through HTML attributes.

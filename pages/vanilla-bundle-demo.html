web-tools

The globals that vanilla-bundle.js installs — terse, framework-free DOM shorthand. No behavior of its own; each helper is a more direct way to write something you'd otherwise spell out with querySelector, loops, and addEventListener. Each block below shows the helper, a representative call, and the result it produced live on this page.

Select + read

ea(sel, cb?) — querySelectorAll → map

Selects all matches and maps each through cb (default: the element itself), returning a real array. Paired with el it reads a property off every match. 

const names = ea('#ea-list li', el.textContent)
// → ["alpha", "beta", "gamma"]

el — property-accessor proxy

el.prop is shorthand for the callback node => node.prop. It exists to feed ea (and other map-callbacks) without writing an arrow — see the el.textContent above.

Lookup — populate variables from the DOM

ids — by id, document-global, live

ids.fooBar resolves getElementById('foo-bar') on access. Because a Proxy turns property reads into lookups, destructuring is keyed by name, not by position — so it can't silently bind the wrong element if the markup is reordered. 

const { idsOut } = ids   // ids.idsOut → getElementById('ids-out')
idsOut.textContent = '…'

ui(root?) — by data-ui, scoped, live

The scoped sibling of ids. ui(root).fooBar finds [data-ui="foo-bar"] within root. Since data-ui carries no uniqueness rule, the same name can repeat across instances — the thing ids can't do. Both cards below use the identical data-ui="count", each scoped to its own card. 

ea('.ui-card', card => {
  const { count, inc } = ui(card)        // scoped to THIS card
  let n = 0
  on(inc, 'click', () => count.textContent = ++n)
})

grab(root?) — by id, scoped, eager snapshot

One scan of a subtree into a plain object, keyed by camelCased id. Unlike ids it's a snapshot taken once (won't see later-added ids), but it hands you the whole bag in a single call. Scoped to #grab-scope below, it sees only that subtree's ids. 

const bag = grab(document.querySelector('#grab-scope'))
// → { firstName: <span>, lastName: <span> }

Structural write

fill(root, spec) — selector → value map

A declarative batch write: for each selector: value pair, every match gets the value applied. The value's type picks the action — string sets text, object is Object.assigned, a function runs on the element, an array runs each callback in turn. 

fill(target, {
  '.t-string': 'string → textContent',
  '.t-object': { textContent: '…', className: 'text-success' },
  '.t-fn':     node => node.textContent = node.tagName,
  '.t-array':  [ cls('font-bold', true), attr('title', '…') ]
})

attr · cls · listen · data — callback helpers

The writes that aren't a plain property set. Each returns an el => el callback, so they slot straight into a fill array (or any ea map). Here one button is styled (cls), seeded with dataset (data), and wired (listen) in a single spec. 

fill(document, {
  '#cb-btn': [
    cls('btn-primary', true),
    data({ role: 'demo', clicks: '0' }),
    listen('click', e => { /* read e.currentTarget.dataset … */ })
  ]
})

Templates

tpl.html / append / prepend / replace — rows → markup → place

A tagged template that builds one string from an array of rows, then a placement step. Interpolations that are functions receive each row; the four verbs differ only in how the result lands (innerHTML, append, prepend, replaceChildren). Call shape is tpl.html`…`(rows)(targetEl). 

const rows = [{ icon: 'rocket-launch', label: 'launch' }, …]

tpl.html`

Events

on(root, type, fn) — addEventListener, returns root

A thin addEventListener that returns the element, so it chains. Use it when you have a single, known target. 

on(ids.onBtn, 'click', () => { /* … */ })

route(root, type, routes) — delegated dispatch

One listener on a container fans out by selector: on each event it walks e.target.closest(sel) through the routes in order and calls the first match's handler with (event, matchedEl). Add or remove children freely — no re-binding. All three controls below share one listener. 

route(ids.routeTarget, 'click', {
  '[data-act="add"]':   () => …,
  '[data-act="clear"]': () => …,
  '.route-item':        (e, match) => … match.textContent …
})

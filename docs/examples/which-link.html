Which link do I send?

Four ways to hand over a rendered page from a web-tools session. The right one depends on where the page lives and where the link gets opened.

Pick the situation

Where does the page live?
Who opens the link?
⭐ Deployed URL. The canonical github.io page works for every viewer. For lib or dist work on a branch, add ?use=<sha>.
🥏 Toss #gh=. A live pointer: the renderer fetches the branch file with the ghToken in that browser's localStorage. Always current, owner only.
Artifact. The app's in-app browser has no localStorage token, so #gh= fails there. The app is signed in to claude.ai, so a baked artifact renders. #gz= is the no-build fallback when the page is small and its CDN loads are fine.
🥏 Toss #gz=. The page rides the link's fragment; nothing is hosted and no token is needed. Artifacts are private to the author on Pro and Max, so they do not travel.
